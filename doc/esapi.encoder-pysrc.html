 74          """
 
 75          Canonicalization is simply the operation of reducing a possibly encoded
 
 76          string down to its simplest form. This is important, because attackers
 
 77          frequently use encoding to change their input in a way that will bypass
 
 78          validation filters, but still be interpreted properly by the target of
 
 79          the attack. Note that data encoded more than once is not something that a
 
 80          normal user would generate and should be regarded as an attack.
 
 81  
 
 82          Everyone U{says<http://cwe.mitre.org/data/definitions/180.html>} you shouldn't do validation
 
 83          without canonicalizing the data first. This is easier said than done. The canonicalize method can
 
 84          be used to simplify just about any input down to its most basic form. Note that canonicalize doesn't
 
 85          handle Unicode issues, it focuses on higher level encoding and escaping schemes. In addition to simple
 
 86          decoding, canonicalize also handles:
 
 87              - Perverse but legal variants of escaping schemes
 
 88              - Multiple escaping (%2526 or &#x26;lt;)
 
 89              - Mixed escaping (%26lt;)
 
 90              - Nested escaping (%%316 or &%6ct;)
 
 91              - All combinations of multiple, mixed, and nested encoding/escaping (%2&#x35;3c or &#x2526gt;)
 
 92  
 
 93          Using canonicalize is simple. The default is just...
 
 94              
 
 95              >>> clean = ESAPI.encoder().canonicalize( request.getParameter("input"))
 
 96              
 
 97          You need to decode untrusted data so that it's safe for ANY downstream interpreter or decoder. For
 
 98          example, if your data goes into a Windows command shell, then into a database, and then to a browser,
 
 99          you're going to need to decode for all of those systems. You can build a custom encoder to canonicalize
 
100          for your application like this...
 
101         
 
102              >>> codeclist = [WindowsCodec(), MySQLCodec(), PercentCodec()]
 
103              >>> encoder = DefaultEncoder( codeclist )
 
104              >>> clean = encoder.canonicalize( request.getParameter( "input" ))
 
105  
 
106          In ESAPI, the Validator uses the canonicalize method before it does validation.  So all you need to
 
107          do is to validate as normal and you'll be protected against a host of encoded attacks.
 
108  
 
109              >>> input = request.getParameter( "name" )
 
110              >>> name = ESAPI.validator().is_valid_input( "test", input, "FirstName", 20, False)
 
111  
 
112          However, the default canonicalize() method only decodes HTMLEntity, percent (URL) encoding, and JavaScript
 
113          encoding. If you'd like to use a custom canonicalizer with your validator, that's pretty easy too.
 
114  
 
115              >>> #... setup custom encoder as above
 
116              >>> validator = DefaultValidator( encoder )
 
117              >>> input = request.getParameter( "name" )
 
118              >>> name = validator.is_valid_input( "test", input, "name", 20, False)
 
119  
 
120          Although ESAPI is able to canonicalize multiple, mixed, or nested encoding, it's safer to not accept
 
121          this stuff in the first place. In ESAPI, the default is "strict" mode that throws an IntrusionException
 
122          if it receives anything not single-encoded with a single scheme.  Currently this is not configurable
 
123          in ESAPI.properties, but it probably should be.  Even if you disable "strict" mode, you'll still get
 
124          warning messages in the log about each multiple encoding and mixed encoding received.
 
125  
 
126              >>> # disabling strict mode to allow mixed encoding
 
127              >>> url = ESAPI.encoder().canonicalize( request.getParameter("url"), False)
 
128  
 
129          @see: U{W3C specifications<http://www.w3.org/TR/html4/interact/forms.html#h-17.13.4>}
 
130  
 
131          @param input_: the text to canonicalize
 
132          @param strict: true (default) if checking for double encoding is 
 
133              desired, false otherwise
 
134  
 
135          @return: a string containing the canonicalized text
 
136  
 
137          @raises EncodingException: if canonicalization fails
 
138          """ 
139          raise NotImplementedError()
