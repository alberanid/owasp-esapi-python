 15 -class SecurityConfiguration: 

 16      """ 
 17      The SecurityConfiguration interface stores all configuration information 
 18      that directs the behavior of the ESAPI implementation. 
 19      <br><br> 
 20      Protection of this configuration information is critical to the secure 
 21      operation of the application using the ESAPI. You should use operating system 
 22      access controls to limit access to wherever the configuration information is 
 23      stored. 
 24      <br><br> 
 25      Please note that adding another layer of encryption does not make the 
 26      attackers job much more difficult. Somewhere there must be a master "secret" 
 27      that is stored unencrypted on the application platform. Creating another 
 28      layer of indirection doesn't provide any real additional security. Its up to the 
 29      reference implementation to decide whether this file should be encrypted or not. 
 30      The ESAPI reference implementation (DefaultSecurityConfiguration.java) does not encrypt 
 31      its properties file. 
 32       
 33      @author Craig Younkins (craig.younkins@owasp.org) 
 34      @since June 3, 2009 
 35      """
