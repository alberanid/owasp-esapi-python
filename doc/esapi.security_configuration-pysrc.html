 20 -class SecurityConfiguration: 

 21      """
 
 22      The SecurityConfiguration interface stores all configuration information
 
 23      that directs the behavior of the ESAPI implementation.
 
 24      
 
 25      Protection of this configuration information is critical to the secure
 
 26      operation of the application using the ESAPI. You should use operating 
 
 27      system access controls to limit access to wherever the configuration 
 
 28      information is stored.
 
 29      
 
 30      Please note that adding another layer of encryption does not make the
 
 31      attackers job much more difficult. Somewhere there must be a master 
 
 32      "secret" that is stored unencrypted on the application platform. Creating 
 
 33      another layer of indirection doesn't provide any real additional security. 
 
 34      Its up to the reference implementation to decide whether this file should 
 
 35      be encrypted or not. The ESAPI reference implementation 
 
 36      (default_security_configuration.py) does not encrypt its properties file.
 
 37      
 
 38      @author: Craig Younkins (craig.younkins@owasp.org)
