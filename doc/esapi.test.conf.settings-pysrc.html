176  # ESAPI Validation 
177  # 
178  # The ESAPI validator does many security checks on input, such as canonicalization 
179  # and whitelist validation. Note that all of these validation rules are applied *after* 
180  # canonicalization. Double-encoded characters (even with different encodings involved, 
181  # are never allowed. 
182  # 
183  # To use: 
184  # 
185  # First set up a pattern below. You can choose any name you want, prefixed by the word 
186  # "Validation." For example: 
187  #   Validation.Email=^[A-Za-z0-9._%-]+@[A-Za-z0-9.-]+\\.[a-zA-Z]{2,4}$ 
188  #  
189  # Then you can validate in your code against the pattern like this: 
190  #   Validator.getInstance().getValidDataFromBrowser( "Email", input ); 
191  #   Validator.getInstance().isValidDataFromBrowser( "Email", input ); 
192  #
